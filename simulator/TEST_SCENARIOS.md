## 1. 인증 테스트

### TC-AUTH-01: 로그인 성공
**우선순위**: 🔴 High
**목적**: 사용자가 올바른 이메일/비밀번호로 로그인할 수 있는지 확인

**전제조건**:
- 테스트 계정: `test1@mail.com` / `123456`

**테스트 단계**:
1. 로그인 페이지로 이동
2. 이메일 입력: `test1@mail.com`
3. 비밀번호 입력: `123456`
4. 로그인 버튼 클릭

**예상 결과**:
- ✅ 메인 대시보드로 리디렉션
- ✅ 사용자 포인트 표시 (1000pt 이상)
- ✅ 프로필 아이콘 표시

---

### TC-AUTH-02: 로그인 실패 (잘못된 자격증명)

3. 비밀번호 입력: `wrongpass`
4. 로그인 버튼 클릭

**예상 결과**:
- ✅ 에러 메시지 표시
- ✅ 로그인 페이지에 머무름

### TC-AUTH-03: 로그아웃

## 2. 예측 제출 테스트

### TC-PRED-01: 기본 예측 제출 (UP)
**우선순위**: 🔴 High
**목적**: 사용자가 UP 방향 예측을 제출할 수 있는지 확인

**전제조건**:
- 로그인 완료
- 최소 100 포인트 보유

**테스트 단계**:
1. 자산 선택: BTC
2. 시간대 선택: 1m
3. 방향 선택: UP
4. 목표 변동률 선택: 0.5%
5. 베팅 금액 입력: 50
6. "Place Prediction" 버튼 클릭

- ✅ 성공 토스트 메시지 표시
- ✅ 포인트 차감 (50pt)
- ✅ 예측 히스토리에 추가
- ✅ 상태: "pending"

## 4. 포인트 및 보상 테스트

### TC-REWARD-01: 승리 시 보상 계산
**우선순위**: 🔴 High
**목적**: 방향 예측 성공 시 올바른 보상 지급

**전제조건**:
- Mock 데이터로 WIN 상태 시뮬레이션

**예상 결과**:
- ✅ 보상 = 베팅액 × 0.8 × 시간대 배율 × 하우스 엣지(0.95)
- ✅ 포인트 증가
- ✅ 통계 업데이트 (total_wins +1)

---

### TC-REWARD-02: 목표 달성 보너스
**우선순위**: 🔴 High
**목적**: 목표 변동률 달성 시 추가 보너스 지급

**예상 보너스**:
- 0.5% → +20pt
- 1.0% → +40pt
- 1.5% → +70pt
- 2.0% → +120pt

**예상 결과**:
- ✅ 방향 보상 + 목표 보너스 합산
- ✅ 연속 승리 배율 적용

---

### TC-REWARD-03: 연속 승리 배율
**우선순위**: 🟡 Medium
**목적**: 연속 승리 시 배율 증가 (최대 2.5x)

- 2연승: 1.2x
- 3연승: 1.5x
- 4연승: 2.0x
- 5연승+: 2.5x

**예상 결과**:
- ✅ 연속 승리 카운트 표시
- ✅ 배율에 따른 보상 증가

### TC-REWARD-04: 패배 시 전액 몰수
